## [RFC 9849 标准化 TLS Encrypted Client Hello 提升隐私](https://www.rfc-editor.org/rfc/rfc9849.html) ⭐️ 9.0/10

IETF 正式发布了 RFC 9849，标准化了 TLS Encrypted Client Hello (ECH) 机制以加密握手元数据。这一最终版本经过多年的草案阶段，使客户端能够利用服务器公钥向网络观察者隐藏 Server Name Indication (SNI)。 这一里程碑通过防止 ISP 和网络中间商在 TLS 握手期间看到用户访问的具体网站，显著增强了用户隐私。然而，它也破坏了依赖 JA3 和 JA4 哈希等 TLS fingerprinting 技术的现有 bot detection 系统。 实现通常依赖 RFC 9848 通过 DNS HTTPS 记录进行 bootstrapping，正如 Caddy 和 Firefox 等工具中所见。虽然它能防止被动监视，但批评者指出加密 ClientHello 移除了区分 scrapers 和真实浏览器的关键信号层。

hackernews · P_qRs · Mar 4, 07:25

**背景**: 在标准的 TLS 握手中，ClientHello 消息以明文发送，揭示了指示目标网站的 Server Name Indication (SNI)。之前的努力如 Encrypted Server Name Indication (ESNI) 是草案，后来演变为 ECH 以更好地处理密钥分发和部署场景。理解此协议需要知道 TLS 虽建立安全连接，但传统上在加密完全建立前会泄露元数据。
